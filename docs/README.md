# Project Documentation

This folder is organized by the kind of decision or knowledge each document captures.

## Structure

```text
docs/
├── roadmap/       # Time-ordered plans, milestones, phase summaries, and progress notes
├── features/      # Product/gameplay feature descriptions and future feature ideas
├── architecture/  # Technical architecture, code organization, and implementation patterns
├── how-tos/       # Step-by-step tutorials for running or operating the project
└── archive/       # Legacy or deprecated project context kept for reference
```

## Roadmap convention

Roadmap files are prefixed with a number so project advancements stay ordered by time:

- `00-*`: initial plans and project direction
- `01-*`, `02-*`, ...: completed phases, milestone summaries, or major progress updates

When adding a new progress document, use the next available number and a short kebab-case name.
